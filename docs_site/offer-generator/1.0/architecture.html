Architecture & Overview

The Offer Generator is a FastAPI + SQLite application that turns equipment parameters into priced Bills of Material, Word/PDF offers, and multi-sheet costing Excels.

Technology stack

  • Backend: FastAPI (Python), served by Uvicorn.

  • Database: SQLite (vlph.db) — committed to the repo (see Deployment & Operations).

  • Documents: docxtpl (Jinja-in-Word templating), docxcompose (merge), openpyxl (Excel), reportlab/LibreOffice (PDF).

  • Frontend: server-rendered static HTML pages (one per equipment) with vanilla JS.

  • Integrations: Google Drive upload (google-api-python-client).

Request flow

  1. User opens an equipment form (e.g. /costing for VLPH).

  2. Form posts parameters to a calculation endpoint (e.g. /api/vlph-calculate).

  3. The endpoint runs the calculation layer → selection engine → BOM builder, producing a priced BOM grouped by MEDIA.

  4. A cost summary (bought-out vs ENCON items, markup) is computed.

  5. On generate, the BOM + customer data fill a Word template; PDF/Excel are derived.

Layered structure

Layer Location

Calculations (sizing/engineering)

calculations/

Selection engine (pick catalog parts)

bom/selectors/

BOM builders (priced line items)

bom/*_builder.py

Pricing / price master

bom/price_master.py, component_price_master table

Web app (routes, endpoints, generation)

main.py

Document engine (docx/pdf/drive)

engine/

Forms (UI)

*_costing.html, combined_builder.html, dashboard.html

Key concepts

  • BOM MEDIA grouping — rows are tagged by system (COMB AIR, gas line, ENCON ITEMS…); drives the grouped Scope of Supply.

  • Markup — config.MARKUP (1.8) applied to bought-out items in the grand total.

  • Templates — cloned from Offer_Template.docx; per-product variants built by build_*_template_from_vlph.py.
