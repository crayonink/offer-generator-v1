Deployment & Operations

Hosting

The app is deployed on Railway (auto-deploys from the main branch on push).

Configuration files

File Purpose

Procfile

process/start command

nixpacks.toml

build configuration

railway.toml

Railway service config

config.py

app constants (MARKUP = 1.8, USD_RATE, …)

requirements.txt

Python dependencies

Database

The SQLite database vlph.db is committed to the repo.

DB edits do not reach Railway until the .db file itself is committed and pushed in the same change. Always commit vlph.db alongside related code.

Pricebook updates

vlph.db carries manual rate edits, so a new price list must be delta-applied, not full-reset, to preserve them.

Local development

  1. Create a virtual environment and install dependencies:

    python -m venv .venv
.venv\Scripts\pip install -r requirements.txt        # Windows
# source .venv/bin/activate && pip install -r requirements.txt   # macOS / Linux

  2. Run the app with auto-reload:

    .venv\Scripts\uvicorn main:app --reload

    Then open http://localhost:8000 (dashboard) — or http://localhost:8000/docs for this handbook.

A plain uvicorn main:app process does not hot-reload Python changes. Pass --reload during development, or restart the process after editing .py files.

The SQLite database vlph.db is read from the repo root. High-fidelity PDF conversion (/api/pdf-quote) needs LibreOffice installed locally; on Railway it is provided via nixpacks.toml.
