Pricelist / Rates

The pricelist is the master list of what every component costs. It is the single place where rates are kept, and every offer the system produces gets its prices from here. Change a rate on this screen and the next offer uses the new figure.

What it is

There are three parts to it:

  • The master rate list — a table that holds one entry for each component, with its price and its supplier.

  • The pricelist screen — where you view and edit the rates. It has a Rates tab and a Raw Material tab, plus several read-only sheets for specific products.

  • The link to offers — when an offer is built, every item looks up its price in this list. So the pricelist is the one place to keep prices up to date.

In addition to this Rates list, the pricelist screen has separate read-only sheets for whole-product prices — blowers, oil / gas / dual-fuel burners, heating & pumping units, recuperator rates, and the radiant-heat element tables — which are used directly when those products are offered.

What is stored for each item

Each entry holds:

  • Item — the component’s name. This name is how an offer finds the price, so it must match exactly.

  • Category — the group it appears under on the screen (for example "Bought Out", "Raw Material", "Motorised Control Valve"). This only controls where it shows on screen — it does not affect pricing.

  • Unit — how it’s sold (per piece, per kilogram, per metre).

  • Price — the current rate.

  • Previous price — the rate before the last edit, used to show the change.

  • Supplier (make) — who supplies it.

  • Model / specification — the model or size code, shown in its own column.

  • Last updated — when the rate was last changed.

A given supplier can hold only one rate per item — the same item name can appear more than once only if it’s under different suppliers (for example the same valve from two makers).

On screen, the item name, its size and its model are shown in separate columns; the system works these out automatically from the stored name and specification.

Reading the screen

  • Items are grouped under category headings. Within a group they are usually ordered by size.

  • Where a category lists more than one supplier, each supplier’s items sit under their own sub-heading.

  • Long or coded names are shortened to a readable label; hovering shows the full name.

  • The columns are: Item, Size, Model / Category, Supplier, Price, Previous Price, Change, and Last Updated.

  • The column headings stay pinned at the top, and the Item column stays visible, as you scroll.

  • A search box filters the list, and duplicate-looking rows are flagged with a warning mark.

Editing a rate

  • Click a price to change it. When you save, the new rate is stored, the old rate is kept as the "previous price" (so the Change column is always measured from the value before your edit), and a brief "recalculating" message appears while the change is pushed through.

  • Click a supplier name to change who supplies the item.

When you save a rate, the change cascades — it is automatically pushed into the related product tables that use that component, so the burners, pumping units and other assemblies all pick up the new figure. That is what the line at the top of the screen means: "Editable rates — changes cascade across every BOM table on save."

How each price is built

Not every price is worked out the same way. There are five kinds:

1. Flat catalogue rates (most of the list). Valves, regulators, solenoids, filters, hoses, gauges, pilot burners and so on are simply stored at the supplier’s rate — there is no calculation. When an offer needs one, the system picks the row that matches the required pipe size (NB). Example: a 80 NB ball valve is taken at its stored rate; no maths involved.

2. Per-kilogram rates. Fabrication and steelwork are priced by weight × a rate:

Item Rate

Fabricated structure

about ₹70 per kg

Air–gas pipeline

₹125 per kg

Stock structural steel (channels, angles)

about ₹60 per kg

Recuperator tubes — stainless steel

₹250 per kg

Recuperator tubes — mild steel

₹70 per kg

Example: a 1,000 kg air–gas pipeline = 1,000 × ₹125 = ₹1,25,000.

3. Flow- or size-selected parts with their own rate. A couple of items are chosen by the duty and then take that row’s price:

  • Gas train — the system picks the gas train whose flow range covers the required gas flow, and takes its price.

  • Air–gas regulator — chosen by pipe size, connection type (threaded or flanged) and turndown ratio, and takes the matching row’s price.

4. Built-up whole-assembly prices.

  • Heating & Pumping Unit — add up the catalogue cost of all its parts and multiply by 1.8. Example: parts ₹1,00,000 → unit price ₹1,80,000.

  • Pumping Unit — the same skid without the heater; its price is already stored at the selling figure (it does not get the ×1.8 again).

  • Blower — the stored catalogue price, including the motor.

  • Burner — the stored catalogue price of the burner set. (Where a GAIL burner is built up from parts, the fabrication cost is multiplied by 1.25.)

5. Whole-offer mark-ups and adjustments. When a complete offer is assembled:

  • in a ladle-preheater or tundish offer, the bought-out items are multiplied by 1.8 before the ENCON-made items are added;

  • a recuperator’s total cost is multiplied by 1.8 (cost-to-sale);

  • then packaging & forwarding, designing and negotiation are added as percentages (they add to the price), transport is added, and the result is rounded to the nearest ₹1,000.

Each product page shows a full worked example of this chain.

How a rate becomes a price in an offer

When an offer is built, each item looks up its price by name in the master list:

  • If the name matches exactly, that price is used.

  • A couple of items (the air–gas regulator and the gas train) are matched by rebuilding their full name from the size and type; if a matching rate exists it is used, otherwise the system falls back to its built-in list price.

Important — the "zero-price" trap. Because the match is by exact name, if an item’s name in the offer differs from its name in the rate list by even one character, it will price at zero — quietly, without an error. So after any change to prices or to item names, it’s worth generating an offer for each fuel and each mode to make sure no line has silently fallen to zero. The fix is always to make the two names match.

Keeping it up to date

  • The price list travels with the system. The rate data lives in a file that is part of the application. An edit made on screen only reaches the live site once that file is saved and published — so price changes must be pushed out, not just made locally.

  • Taking in a new pricebook. When a new pricebook spreadsheet arrives, there is a careful "update only what changed" method that compares the old and new books and applies only the differences — so any rates that were hand-edited are preserved. A full re-import is also possible, but it overwrites everything and does not protect manual edits, so the careful method is preferred.
