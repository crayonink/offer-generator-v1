= Vertical Ladle Preheater (VLPH)
:description: Inputs, outputs and calculation flow for the Vertical Ladle Preheater offer.

Module: `vlph`. Form: `/quote` → `vlph_costing.html`. Status: ✅ Complete (see xref:product-scope.adoc[Product Scope & Roadmap]).

== Inputs

The calculation form (`/costing` → `vlph_costing.html`) posts to `/api/vlph-calculate` (`main.py:1373`) using the `VLPHCalcRequest` model (`main.py:636`). The finished-offer form posts to `/api/generate-quote` (`main.py:4445`) using the `QuoteRequest` model (`main.py:684`). The two are split: `VLPHCalcRequest` drives the BOM/costing math; `QuoteRequest` carries customer data, scope toggles and the single all-inclusive price (built client-side) into the document.

== Outputs

`/api/vlph-calculate` returns a JSON object (`main.py:1613`) with `calculations`, `pipes`, `equipment`, the full `bom` line list, and `cost_summary` (`bought_out_total`, `encon_total`, `grand_total`). `/api/generate-quote` returns download / preview / PDF URLs and a price summary, and writes the offer documents.

=== BOM line items

Each row is `(MEDIA, ITEM NAME, REFERENCE, QTY, MAKE, UNIT PRICE, TOTAL)` built by `_row` (`vlph_builder.py:50`). Rows are grouped by `MEDIA`:

* *COMB AIR* — compensator, pressure gauge with TNV, pressure switch low, (PLC/P.PID: orifice plate + DPT), control valve, butterfly valve (L&T), rotary joint; plus pilot/UV ball valves + flexible hoses when auto-ignition is on.
* *{FUEL} LINE* — gas-train package (NG/RLNG/LPG via MADAS) *or* discrete-component trains for Mixed Gas (`_mix_gas_line_rows`), COG (`_cog_line_rows`), BFG (`_bfg_line_rows`); oil line is gated to 20 NB (ball valve, flowmeter, motorised control valve, solenoid). AGR appears in `manual` / `plc_agr` / `pid`.
* *PURGING LINE* — nitrogen block when `purging_line == "yes"` (`vlph_builder.py:923`).
* *{LPG/NG/COG} PILOT LINE* — when auto-ignition / pilot included.
* *ENCON ITEMS* — burner (Gas/Oil/Dual), bearing+plummer+shaft *or* SWIVEL ASSEMBLY, fabrication/structure, AIR-GAS PIPELINE, blower, ignition transformer / sequence controller / UV sensor / pilot burner (auto-ignition), HPU/pumping unit (oil), ceramic fibre.
* *MISC ITEMS* — static items (`bom/static_items.py`) filtered by mode (control panel, instrument ball valves, PLC/PPID/PID, ratio controller, temperature transmitter, hydraulic power pack), plus CONTROL PANEL and PLC WITH HMI.

Three summary rows are appended: *BOUGHT OUT ITEMS*, *ENCON ITEMS*, *GRAND TOTAL* (`vlph_builder.py:1148`). "Bought out" = every non-`ENCON ITEMS` row (excluding `RATIO CONTROLLER`); "ENCON" = `MEDIA == "ENCON ITEMS"` rows.

=== Scope of supply / annexures

`generate_quote_docx` (`engine/quote_writer.py:677`) fills `Offer_Template.docx` (`quote_writer.py:5`). It produces:

* *Annexure I* — Scope of Supply, with dynamic Vertical / Horizontal columns and `qty_label_*` set headers; the unused column is post-stripped (`quote_writer.py:1064`). Scope strings adapt to hood type, HPU/oil, pilot/auto-ignition, and control mode.
* *Annexure III* — Price schedule: one equipment row bound to `*_vertical`, `qty × unit price = total`, then a *Transport* line (when > 0) inserted before *TOTAL* by `_break_out_transport` (`main.py:4375`).
* *Annexure IV* — editable Terms & Conditions from the `tnc_*` fields.
* A MAKE LIST table built from `bom_items` on the final page.

=== Output documents

The route writes three files into `QUOTES_FOLDER` named `{YYYY-MM-DD}_{Company}_{VLPH}-{tons}T.{ext}` (`main.py:4559`):

. `.docx` — the offer (from `Offer_Template.docx`).
. `.pdf` — LibreOffice render of the docx, with a reportlab fallback (`main.py:4573`).
. `.json` — persisted `quote_data` for later PDF regeneration.

The offer is also mirrored to Google Drive (async) and logged to the `quotes_log` table (`main.py:4602`).

== Calculation Flow

. *Resolve fuel & mode* (`main.py:1388`). `f1_cv = fuel1_cv if > 0 else fuel_cv`. Dual-fuel active when `fuel2_type != "none"` and `fuel2_cv > 0`. `n_burners = max(1, num_burners)`.

. *Compute fuel & air flow.*
* *Direct mode* (`main.py:1396`): `heat_kcal_hr = direct_heat_input_kcal_hr` if given, else `direct_burner_capacity × 860 × n_burners`. Then `ng_flow = heat_kcal_hr / f1_cv` and `air_flow = heat_kcal_hr × 118 / 100000`.
* *Calc mode* (`main.py:1407`): `calculate_burner` (`calculations/burner.py:35`). `avg_temp_rise = (Tf−Ti)/2 + 50` when `Ti > 100` else `(Tf+Ti)/2`; `firing_rate = refractory_weight × heat_factor × avg_temp_rise`; `heat_load = firing_rate / efficiency`; `fuel_consumption = heat_load / fuel_cv`; `÷ time_taken_hr` then `× 1.1` (10 % margin) → `extra_firing_rate_nm3hr` (= `ng_flow`); `air_qty = fuel_cv × extra_firing_rate × 118 / 100000` (= `air_flow`).

. *Burner pressure* derived from blower pressure: 28→24 WG, 40→36 WG (`main.py:1422`).

. *Per-burner split* (tundish, `main.py:1426`): blower / gas-train / main line size on total flow; burner sizes on `ng_flow / n_burners`. For dual-fuel, blower air = `max(fuel1_air, fuel2_air)` (`main.py:1432`); oil fuel-2 LPH pre-computed for blower CFM.

. *Equipment selection* — `select_equipment` (`selection_engine.py:20`):
* *Gas pipe NB* via `calculate_pipe_sizes` at `ng_velocity = 12.7 m/s` for all gas fuels (`pipes.py:33`); area = flow/velocity, diameter rounded up to the next `STANDARD_PIPE_NB` (`pipes.py:13`). Oil line NB is flow-based: ≤250 LPH → 20 NB else 25 NB (`select_oil_pipe_nb`, `pipes.py:37`).
* *Burner* — `select_encon_mg_burner`, pricelist section keyed by `dual`/`oil`/`gas` and CV, `burner_pressure_wg` (`selection_engine.py:44`).
* *Gas train* — `select_ng_gas_train` picks the row whose `[min_flow, max_flow]` brackets `ng_flow`, largest if over-range; price prefers the editable `component_price_master` "Gas Train {in}x{out}" row (`ng_gas_train.py:8`).
* *AGR* — only when needed (manual / plc_agr / pid). Sized from the *gas-train outlet NB* (not raw pipe NB), Flanged if NB ≥ 65 else Threaded (`selection_engine.py:68`).
* *Blower* — CFM logic: `gas_cfm = air_flow / 1.7`; `oil_cfm = equivalent_lph × 10` (atomisation air, dual-fuel takes the max); `required_hp = cfm × pressure_in_wg / 3200`; `select_blower` picks the smallest HP ≥ required in the chosen `ENCON {series}/%` series (`blower_selector.py:20`).
* *Air line NB (the key rule)* — sized from the *selected blower's CFM × 1.7* back to Nm³/hr, run through the pipe formula, then floored: `air_line_nb = max(125, cfm-based NB, air_duct NB)` (`selection_engine.py:104-111`). Butterfly valve (lever→gear fallback) and rotary joint size to this NB.
* *Air duct / motorised control valve* sized from `air_flow` (`selection_engine.py:102`).
* *HPU / Pumping Unit* — oil fuels only; LSHS/FO use a standalone Pumping Unit, others use HPU, sized on `equivalent_lph` (`selection_engine.py:122`).

. *Re-size pipes for display* with `air_flow_nm3hr = equip1["air_line_flow"]` (cfm×1.7) so the air line shown matches the BOM (`main.py:1477`). Tundish re-selects the burner for per-burner flow (`main.py:1483`).

. *Auto-fill structure & ceramic* from `fabrication_ladle_mapping` via `lookup_ladle_fab_pipeline` (nearest ladle capacity for the hood type, `vlph_builder.py:216`). User overrides win; pipeline kg stays user-supplied (`main.py:1549`).

. *Build BOM DataFrame* — `build_vlph_manual_df` (control_mode `manual`, `vlph_builder.py:1166`) or `build_vlph_120t_df` (automatic, `vlph_builder.py:806`). Manual passes `base_only=True` to `_fuel_line_rows`, stripping AGR / orifice / DPT / control valve.

. *Per-row pricing* — `_row` (`vlph_builder.py:50`): if `unit_price_override` is supplied it is used verbatim (computed prices such as `pipeline_weight_kg × PIPELINE RATE`, `ms_structure_kg × FABRICATION RATE`, selector prices, or NB-scaled DB lookups). Otherwise `get_price(item)` does an *exact-match* DB lookup (`_get_price_fuzzy`, no fuzzy fallback). *Zero-price behaviour:* an unmatched item name prices to 0 and logs `WARNING: No price found for '{item}'` — i.e. a row whose item name does not equal a DB key silently contributes 0. Blank `MAKE` defaults to `ENCON` (`vlph_builder.py:65`).

. *Tundish multiplication* (`vlph_builder.py:1107`): per-burner items and `* PILOT LINE` media rows get `QTY` and `TOTAL` × `num_burners`; main equipment stays at qty 1.

. *Summary roll-up* (`vlph_builder.py:1137`): `bought_out_total` = non-ENCON rows (excluding `RATIO CONTROLLER`); `encon_total` = ENCON rows; `grand_total = bought_out + encon`. Appended as the three summary rows.

. *Commercial adjustments (client-side, in `vlph_costing.html`).* `recalcGrandTotal` (`vlph_costing.html:2000`): `Grand Total = bought_out × markup (default 1.8) + encon`. Then `pfAmt`, `designAmt`, `negAmt` each = `Grand × pct/100` and `Final Total = round((Grand + pfAmt + designAmt + negAmt + transAmt) / 1000) × 1000` — P&F, designing and *negotiation are all added* (not subtracted), and the result is rounded to the nearest Rs. 1000 (`vlph_costing.html:2020`). The Final Total is sent to the backend as the single line item's `unit_price` (`vlph_costing.html:2505`); `transport_amt` is sent separately.

. *Quote pricing* — `calculate_quote` (`engine/quote_engine.py:88`) uses the pre-set `unit_price` when > 0, else falls back to a DB sum by product type. GST/freight are not added; `grand_total = subtotal`.

. *Document assembly* — `generate_quote_docx` (`quote_writer.py:677`) renders `Offer_Template.docx` with the mapped context, strips the unused scope column, and saves the `.docx`. Then `_break_out_transport` (`main.py:4375`) reduces the equipment row by `transport_amt` and inserts a *Transport* line before *TOTAL* (TOTAL unchanged, so the schedule reconciles). Finally the PDF and `.json` are written, the files mirror to Drive, and the offer is logged to `quotes_log` (`main.py:4569-4629`).
